# jonathanwrede.de

Personal site for **jonathanwrede.de**. Built with [Astro](https://astro.build),
Tailwind CSS, and a tiny Markdown-driven blog. Deployed by serving the static
build output through a host-level Caddy container.

## Local development

```bash
npm install
npm run dev          # http://localhost:4321
```

Hot-reloads on changes to `.astro`, `.md`, and `tailwind.config.js`.

## Production build

```bash
npm run build        # outputs to ./dist/
npm run preview      # serve dist/ locally to sanity-check
```

`dist/` contains:
- `index.html`, `blog/index.html`, `blog/<slug>/index.html` (one per post)
- `_astro/*.css` (Tailwind, ~40 KB minified)
- Static assets copied from `public/` (CVs, profilbild, robots.txt, etc.)
- `sitemap-index.xml` + `sitemap-0.xml` (auto-generated by `@astrojs/sitemap`)

## Deployment

There is no CI. The site is served directly from the build output on this
host by the Caddy container defined in `~/infra/`.

```
~/infra/Caddyfile             # routes jonathanwrede.de в†’ /srv/jonathanwrede.de
~/infra/docker-compose.yml    # bind-mounts ./dist into the Caddy container
```

The relevant volume mount in `~/infra/docker-compose.yml`:

```yaml
- /home/jonathan/jonathanwrede.de/dist:/srv/jonathanwrede.de:ro
```

Caddy's `try_files {path} {path}/index.html /index.html` resolves directory
URLs (`/blog/`, `/blog/<slug>/`) to their `index.html` and falls back to the
homepage for unknown routes.

### Deploy a change

```bash
# 1. Build
cd ~/jonathanwrede.de
npm run build

# 2. Caddy is already mounted on dist/ and reads files live вЂ” no restart
#    needed unless you change the Caddyfile or docker-compose.yml itself.
```

If you change `~/infra/Caddyfile` or `~/infra/docker-compose.yml`:

```bash
docker compose -f ~/infra/docker-compose.yml up -d
```

### TLS

Certificates are managed automatically by Caddy via Let's Encrypt
(stored in the `caddy-data` volume). No action needed.

## Adding a blog post

1. Create `src/content/blog/<slug>.md` with frontmatter:

   ```md
   ---
   title: "Post title"
   description: "One-sentence summary used for meta description, OG, and listings."
   pubDate: 2026-05-03
   tags: ["llm", "python"]
   lang: "en"          # or "de"
   draft: false        # set true to hide from listing + sitemap
   ---

   Markdown body...
   ```

2. `npm run build`. The new post:
   - appears at `https://jonathanwrede.de/blog/<slug>/`
   - is listed at `https://jonathanwrede.de/blog/`
   - is referenced from the "Latest writing" card on the homepage (top 3)
   - is added to `sitemap-0.xml`
   - emits per-post `BlogPosting` JSON-LD with title, description, pubDate, tags

The schema is enforced in `src/content/config.ts` вЂ” typos in frontmatter fail
the build.
